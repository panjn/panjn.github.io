一文了解静态类型语言与动态类型语言

通常来说，关于这两者的定义，你会看到两种说法。

是否在运行前知道类型

第一种是:

A language is statically typed if the type of a variable is known at compile time.

A language is dynamically typed if the type is associated with run-time values, and not named variables/fields/etc.

如果我们可以在编译期知道变量的类型，我们就说这种语言是静态类型(statically typed)。这里我更倾向于把编译期改为运行期之前(before run-time), 因为有些语言并不编译（后文也一样）。

如果我们运行期才知道变量的类型，则称其为动态类型的(dynamically typed)。

一文了解鸭子类型(Duck typing)

鸭子测试

鸭子测试的意思是，如果我要判断一个东西是否是鸭子，我就看它是不是像鸭子那样走路，像鸭子那样嘎嘎叫，如果是，那它应该就是一只鸭子。（姑且不论这样判断一个东西是否是鸭子是否正确）

如果它看起来像鸭子、游泳像鸭子、叫声像鸭子，那么它可能就是只鸭子。
