一文了解静态类型语言与动态类型语言

通常来说，关于这两者的定义，你会看到两种说法。

是否在运行前知道类型

第一种是:

A language is statically typed if the type of a variable is known at compile time.

A language is dynamically typed if the type is associated with run-time values, and not named variables/fields/etc.

如果我们可以在编译期知道变量的类型，我们就说这种语言是静态类型(statically typed)。这里我更倾向于把编译期改为运行期之前(before run-time), 因为有些语言并不编译（后文也一样）。

如果我们运行期才知道变量的类型，则称其为动态类型的(dynamically typed)。

是否在运行前做类型检查

而第二种说法则将静态类型vs动态类型 等价于 静态类型检查vs动态类型检查：

Statically typed programming languages do type checking (i.e. the process of verifying and enforcing the constraints of types) at compile-time as opposed to run-time.

静态类型语言(statically typed languages)在编译期做类型检查(statically type checking)，而动态类型语言(dynamically typed languages)则在运行期做类型检查(dynamically type checking)。（简单来说，类型检查是检查程序是否符合语言所规定的的一套关于类型的规则，比如1+'0'在有些语言是合法的，而在另一些语言则是不合法的。）

通常来说，这也没什么不妥，如果你在运行前知道了变量的类型，你当然也会把类型检查放在运行之前，不然你就没必要在运行期前知道类型了。道理是这样说，但我还是喜欢第一种定义，因为你可以想象有一种XJB语言， 它就是规定要在运行前知道变量的类型，但又不在运行前做类型检查。
