谈Python的执行过程（1）：认识字节码和CPython对象

本文描述CPython是如何执行一个Python程序的。 通常来说我们运行Python有以下几种方式：

  • 将一个字符串作为python代码运行

  • 运行一个模块

  • 运行一个Python文件

无论是哪种方式，我们都是只是在跑一个可执行程序Python，并附带了一些参数(比如-c "print(1+1))而已。 而Python是一个用 C 语言编写的程序（CPython）。关于Linux下C程序是如何运行的可以看我之前写的文章 一文了解C程序是如何运行起来的。简单来说，这个过程是这样的:

co_consts和co_names字段

在之前的指令中，我们可以看到四个指令的操作数分别是0, 0, 1, 0, 它们分别代表什么? 实际上，

  • 对LOAD_CONST 0指令，0表示的是co_consts中数据的索引；
  • 对STORE_NAME 1指令，1表示的是co_names中数据的索引。

PyLongObject

它的内存布局类似于之前的PyBytesObject, 只不过它的数据元素是PyObject*指针，而PyBytesObject是char, ob_size=2表示这个tuple长度为2，

co_consts的第一个item为1000， 其实它的类型是PyLongObject:

这里为了显示这个PyLongObject,我们做了几层类型转换

  • (PyTupleObject*)co->co_consts 首先将co->co_consts转为PyTupleObject类型的指针
  • (PyTupleObject*)co->co_consts)->ob_item 处的值是两个PyObject的地址
  • *((PyTupleObject*)co->co_consts)->ob_item)取第一个地址，它实际指向的是PyLongObject
  • 最后将这个地址转换为PyLongObject指针，再解引用，便得到了表示1000的PyLongObject。

PyLongObject的内存布局实际也与PyBytesObject类型，只不过它的数据元素是整数(比如uint32_t), 这里它只有一个元素1000， 假如有一个超级大的数，就可以用多个元素来表示它。这就是Python的整数的精度这么大的原因。

CPython中的对象

总结一下，目前为止，我们已经见过了几种类型的对象， PyObject, PyBytesObject, PyTupleObject, PyLongObject，PyCodeObject

再次看之前的PyBytesObject内存布局图，PyObject类型其实就是前四个字段，而后三者都加了个ob_size字段，这样它们都是PyVarObject类型， 即所谓的变长对象。变成并不是说sizeof(PyBytesObject)会得到不固定的结构，C中结构体大小肯定是固定的，只不过利用ob_size， 我们可以把数据紧接着放在结构体后面，这后面数据的大小可以通过ob_size和结构中定义的数据元素类型确定，这样就达到了变长的目的。

  • PyBytesObject的ob_size表示在结构体末尾的内存开始往后存储了多少个char (byte)
  • PyLongObject的ob_size表示在结构体末尾的内存开始往后存储了多少个整数 (digit, 具体类型可以是uint32_t)
  • PyTupleObject的ob_size表示在结构体末尾的内存开始往后存储了多少个PyObject指针。

PyCodeObject没有ob_size字段，它是一个定长对象，并有很多自己的字段，我们在前文中分析了co_code、co_consts和co_names字段。
